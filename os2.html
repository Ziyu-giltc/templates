Installation Guide

Aurora OS can be installed in just a few minutes. Follow this guide to get up and running.

Step 1: Download the ISO

Download the latest Aurora OS ISO from our download page. The file is approximately 2.4 GB.

Step 2: Create Bootable Media

Use one of the following tools to create a bootable USB drive:

  • Etherballoon (Windows) - Recommended for Windows users
  • balenaEtcher (macOS/Linux/Windows) - Cross-platform solution
  • dd (Linux/macOS) - Command-line option
# Linux/macOS dd command
sudo dd if=aurora-os-3.2.1.iso of=/dev/sdX bs=4M status=progress && sync

Step 3: Boot from USB

Insert the USB drive and reboot your computer. Press the appropriate key (usually F12, F2, or Del) to access the boot menu and select your USB drive.

Step 4: Run the Installer

Aurora OS boots into a live environment. Double-click the "Install Aurora OS" icon on the desktop to launch the graphical installer.

Warning: Installing an operating system will erase data on the target drive. Always back up important files before proceeding.

Software Management

Aurora OS uses Flatpak for application packaging, ensuring apps are sandboxed and secure.

Using Aurora Store

The Aurora Store provides a graphical interface for browsing and installing applications. It automatically handles updates and dependencies.

Command Line

For power users, the aurora command provides full control:

# Search for an application
aurora search firefox

# Install an application
aurora install firefox

# Update all applications
aurora update

# Remove an application
aurora remove firefox

System Settings

Customize your Aurora OS experience through the System Settings panel.

Appearance

Change themes, icons, fonts, and window styles. Aurora OS supports both light and dark modes with automatic switching based on time of day.

Displays

Configure multi-monitor setups, resolution, and scaling. HiDPI displays are automatically detected and configured.

Privacy & Security

Manage app permissions, firewall settings, and encryption options. By default, Aurora OS blocks all incoming connections and requires explicit permission for app access.

Troubleshooting

Common issues and their solutions.

Boot Issues

If Aurora OS fails to boot after an update, use the rollback feature:

aurora rollback

Wi-Fi Not Working

Some wireless cards require proprietary drivers. Install them from:

aurora install firmware-drivers

Application Won't Start

Try running the application from the terminal to see error messages:

flatpak run com.example.App
